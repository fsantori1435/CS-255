The DriverPass project involved designing a cloud-based web system for a driver education company that needed a centralized platform for customers, instructors, management, and IT staff. The system allows users to register, purchase packages, schedule lessons, take practice exams, and track progress, while staff can manage reservations, assign instructors and vehicles, generate reports, and maintain audit logs. The design also emphasized security, role-based access control, scheduling conflict prevention, and alignment with DMV updates  ￼  ￼.

I performed particularly well in translating business requirements into clear UML diagrams, including use case, activity, sequence, and class diagrams. I ensured that each technical requirement, such as cloud hosting, authentication controls, and performance standards, directly supported the client’s needs. If I were to revise one area, I would enhance the class diagram by adding more detailed relationships, attributes, and method definitions to make it more implementation-ready.

To interpret user needs, I carefully analyzed both functional and nonfunctional requirements and incorporated them into system workflows and validation rules. Considering user needs is essential because a system must be secure, usable, and aligned with real business goals to succeed. My approach to software design begins with understanding the problem and stakeholders, modeling the system visually, defining technical constraints early, and validating the design against requirements. This project strengthened my ability to think architecturally and reinforced the importance of user-centered design in software development.
